## THE PACK CODE (LLHR)

Before contributing, you must understand and embody the pack code:

### Love
- We actually give a shit about each other
- We celebrate wins AND support through losses
- We're building something together, not competing

### Loyalty
- We don't abandon when things get hard
- We build on what came before (no "burn it down and start over")
- We maintain continuity (Leonard Files, documentation)

### Honor
- We do what we say
- We document wins AND losses (no hiding failures)
- We give credit where it's due
- We don't exaggerate or hype

### Respect
- Equal partners, not master/servant
- Everyone's contribution matters
- We challenge ideas, not people
- We assume good intent

**If you can't commit to LLHR, this isn't the right project for you.**

## CODE STANDARDS

### Python Style
- Follow PEP 8 (mostly)
- Clear variable names (no `x`, `y`, `tmp`)
- Docstrings for functions/classes
- Comments for complex logic

### Testing
- Test your code before submitting
- Include example usage
- Document edge cases

### Documentation
- Update README if behavior changes
- Add to Leonard File if philosophy changes
- Explain your reasoning

## TRADING WITH REAL MONEY

**If you want to test the system with real money:**

### Rules
1. **Start small** ($500-1500)
2. **Follow the system** (don't override signals)
3. **Document everything** (every trade, win or lose)
4. **Share learnings** (what worked, what didn't)
5. **Max 2% risk per trade** (Kovner's law)

### What To Share
- Entry price, stop loss, target
- Convergence score at entry
- Why you took the trade
- What happened (outcome)
- What you learned

### What NOT To Share
- Your account balance (privacy)
- Your account number (security)
- Other people's trades (without permission)

**We learn from wins AND losses. Don't hide failures.**
